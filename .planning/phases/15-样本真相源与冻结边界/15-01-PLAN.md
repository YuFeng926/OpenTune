  <action>
    在 `Source/PluginProcessor.cpp` 实现 Task 1 的四个接口，具体数学规则必须写死如下：

    1. `getClipSampleRangeById(...)` / `getPluginClipSampleRange(...)`
       - 从现有 `tracks_ -> TrackState::AudioClip -> audioBuffer->getNumSamples()` 派生
       - `startSample = 0`
       - `endSampleExclusive = clip.audioBuffer->getNumSamples()`
       - `sampleCount = endSampleExclusive - startSample`
       - `audioBuffer == nullptr` 或 `getNumSamples() <= 0` 时返回 `false`

    2. `freezeRenderBoundaries(...)`
       - 使用 `TimeCoordinate::secondsToSamples(..., TimeCoordinate::kRenderSampleRate)` 把输入 seconds 只换算一次
       - `trueStartSample = clamp(startSample, clipRange.startSample, clipRange.endSampleExclusive)`
       - `trueEndSample = clamp(endSample, trueStartSample, clipRange.endSampleExclusive)`
       - `publishSampleCount = trueEndSample - trueStartSample`
       - `frameCount = max(1, (publishSampleCount + hopSize - 1) / hopSize)`
       - `synthSampleCount = frameCount * hopSize`
       - `synthEndSample = trueStartSample + synthSampleCount`
       - `hopSize <= 0`、`publishSampleCount <= 0`、`clipRange.isValid() == false` 时返回 `false`

    3. `preparePublishedAudioFromSynthesis(...)`
       - 只有 `synthesizedAudio.size() == boundaries.synthSampleCount` 时返回 `true`
       - 成功时只复制前 `boundaries.publishSampleCount` 个 sample 到 `publishedAudio`
       - 失败时清空 `publishedAudio` 并返回 `false`

    约束：
    - 不允许在这些 helper 里把 `frameCount` 再降回 seconds
    - 不允许新增第二套 sample-count 缓存字段；所有 sample span 仍从 clip buffer 真相源派生
  </action>
