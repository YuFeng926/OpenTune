    - `merge(...)` 先对 `reasonsMask` 做按位 OR
    - `priority` 必须取当前 state 与 request 的最大值
    - 只要 `request.fullRepaint == true`，或者 `request.hasDirtyArea == false` 且 state 已有工作，则 state 立即升级为 `fullRepaint = true`、`hasDirtyArea = false`，并把 `dirtyArea` 清空
    - 只有在 `state.fullRepaint == false` 且 `request.hasDirtyArea == true` 时，才允许把 dirty rect 写入/union 到 `state.dirtyArea`
    - `hasWork()` 只有在 `reasonsMask != 0` 时返回 true
    - `clear()` 必须把 `reasonsMask` 归零、`fullRepaint` 归 false、`hasDirtyArea` 归 false、`dirtyArea` 归空矩形、`priority` 重置为 `Background`
    - `makeVisualFlushDecision(...)` 必须：
      - 如果 `state.hasWork()` 为 false，则返回 `shouldRepaint = false`
      - 如果 `state.fullRepaint` 为 true，则返回 `shouldRepaint = true`、`fullRepaint = true`、`hasDirtyArea = false`
      - 如果 `state.hasDirtyArea` 为 true，则先与 `localBounds` 求交；交集非空时返回 dirty repaint；交集为空时退化成 `fullRepaint = true`
      - `priority` 原样透传到 flush decision
  </action>
