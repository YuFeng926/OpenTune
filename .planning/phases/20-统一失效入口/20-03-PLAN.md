<objective>
把 `PianoRollComponent` 与 `PianoRollToolHandler` 的所有视觉入口统一收敛到 `invalidateVisual(...)`，删除旧 `requestInteractiveRepaint()` / `requestRepaint` 语义，并把 `FrameScheduler` 压缩成单一 flush 出口。

Purpose: Phase 20 的目标不是“少几个 repaint 调用”，而是让同一语义变化只存在一个视觉入口、一个待刷状态、一个 flush 出口。只有这样，Phase 21 才能在此基础上继续做单 VBlank 时钟收敛。

Output: 不再暴露旧 repaint API 的 `PianoRollComponent` / `PianoRollToolHandler`，以及统一的 pending invalidation merge + flush wiring。
</objective>
