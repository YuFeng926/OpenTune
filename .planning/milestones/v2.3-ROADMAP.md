# Roadmap: OpenTune v2.3 Source/Materialization/Placement persisted truth

## Overview

`v2.3` 是 `v2.2` 之后的结构收敛 milestone。它不引入新的产品形态，也不改变 Standalone / VST3 双目标定位；它冻结的是当前 live tree 已落地的主线结果：**`Source + Materialization + Placement` persisted truth，`Projection` 只做 derived contract**。

这个 milestone 的核心成果是：shared runtime shell 稳定为 `SourceStore + MaterializationStore + StandaloneArrangement + VST3AraSession`；PianoRoll / playback / Standalone / VST3 ARA 的 public contract 统一回到 `materialization + projection`；split / merge / delete / reclaim 生命周期与 undo lineage 已收敛为唯一正确结构。当前剩余 gap 主要是手工 L5 旅程、macOS bundle inspection，以及已明确后移的 F3/F5 独立任务。

## Status

- **Milestone:** `v2.3`
- **Status:** FROZEN / READY-FOR-RELEASE-CANDIDATE
- **Latest shipped version:** `v2.2`
- **Not yet shipped because:** L5 manual journeys + macOS bundle inspection remain open

## Scope Included

- `AppPreferences` typed persistence、shared/standalone page composition、shared visual preferences 同步
- `AudioEditingScheme` 显式规则化、notes-first / corrected-f0-first 交互收敛
- Standalone-only mac bundle metadata / docs path / help lookup 收敛
- `SourceStore + MaterializationStore + StandaloneArrangement + VST3AraSession` 四段真相 runtime shell 定型
- owner model 从旧 `Content/Placement` 两层假设纠正为 `Source + Materialization + Placement`
- `MaterializationTimelineProjection` / `materialization + projection` public contract 统一
- lineage state machine、retire/revive/sweep reclaim 基础设施
- `PlacementSplitAction` / `PlacementMergeAction` / `PlacementDeleteAction` lineage-aware undo/redo
- VST3 PluginEditor command-path owner-missing invariant signaling

## Deferred / Not In This Milestone

- **F3:** `SourceStore` 接管 hydration 物理 owner（高风险，需专门锁序设计）
- **F5:** reclaim registry 双格式统一（低价值，可选）
- L5 Standalone / VST3 manual journeys
- macOS `.app` bundle inspection
