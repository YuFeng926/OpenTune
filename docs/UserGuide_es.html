1. Introducción al producto

OpenTune es un software de código abierto de corrección de tono con IA basado en un vocoder neuronal. A diferencia de los algoritmos DSP tradicionales que transponen directamente, utiliza el vocoder PC-NSF-HiFiGAN para regenerar la voz preservando los formantes originales, manteniendo un timbre natural y sólido incluso con ajustes extremos de tono.

OpenTune ofrece dos modos de funcionamiento: Standalone (independiente) y VST3/ARA (plugin para DAW). Puede usarse como editor multipista independiente o integrarse en un DAW para edición colaborativa en tiempo real.

Nota sobre aceleración GPU

OpenTune es compatible con aceleración GPU mediante DirectML (Windows) o CoreML (macOS). Si la GPU no está disponible, se revierte automáticamente a CPU sin afectar el uso normal. Puede cambiarse manualmente a "Prioridad CPU" en Preferencias.

3. Instalación e inicio

3.1 Descargar el software

Visite la página de GitHub Releases para descargar la última versión.

3.2 Estructura de archivos

  • OpenTune.exe — Programa principal (Standalone)
  • OpenTune.vst3 — Plugin VST3 (para DAW)
  • onnxruntime.dll / onnxruntime_providers_shared.dll — ONNX Runtime
  • DirectML.dll — DirectML Runtime (aceleración GPU)
  • models/ — Carpeta de modelos IA
    • rmvpe.onnx — Modelo de extracción de tono (~28 MB)
    • pc_nsf_hifigan_44.1k_ONNX/ — Modelo de vocoder
  • docs/UserGuide.html — Esta guía del usuario

3.3 Iniciar el programa

Haga doble clic en OpenTune.exe para iniciar la versión independiente. Los modelos IA se inicializan automáticamente en el primer inicio. Para el plugin VST3, copie OpenTune.vst3 en el directorio de escaneo VST3 de su DAW.

5. Inicio rápido

1

Importar audio

Arrastre y suelte archivos de audio en la ventana del programa, o impórtelos mediante el menú File → Import Audio. Compatible con formatos WAV, MP3, FLAC, OGG, AIFF, entre otros.

2

Extracción automática de tono

Tras la importación, el software extrae automáticamente la curva de tono original, mostrada en rojo en el rollo de piano.

3

Configuración de escala

El software detecta automáticamente la escala, o puede seleccionar manualmente la nota raíz (C~B) y el tipo de escala (Mayor, Menor, Cromática, etc. 8 tipos).

4

Editar tono

Use la herramienta de corrección automática (corrección con un clic) o dibuje manualmente notas/anclajes/curvas a mano alzada. El tono corregido se muestra como una curva azul.

5

Ajustar parámetros

Ajuste Retune Speed, Vibrato Depth/Rate, etc. en el panel de parámetros. Admite configuración de vibrato independiente por nota.

6

Previsualizar y exportar

Presione Espacio para reproducir y previsualizar. Cuando esté satisfecho, exporte mediante File → Export. Admite exportar clip seleccionado, pista individual o mezcla de bus.

6. Funciones principales

6.1 Importación de audio

  • Formatos compatibles: WAV, MP3, FLAC, OGG, AIFF, entre otros
  • Remuestreo automático a 44.1 kHz durante la importación
  • Importación por arrastrar y soltar (directamente a la ventana)
  • Soporte multipista: hasta 12 pistas independientes, cada una con múltiples clips
  • Importación por lotes de múltiples archivos (secuenciales o en pistas separadas)

6.2 Extracción de tono (RMVPE)

  • Tras importar el audio, se extrae automáticamente la curva de tono original (mostrada en rojo)
  • Basado en el modelo de aprendizaje profundo RMVPE, con alta precisión y fuerte resistencia al ruido
  • Admite detección vocal en el rango de frecuencia de 50 Hz – 1100 Hz

6.3 Detección y selección de escala

Detección automática de escala integrada mediante el algoritmo Krumhansl-Schmuckler, compatible con 8 tipos de escala:

MenorMin.Escala menor natural

Menor ArmónicaH.Min.Menor con séptimo grado elevado

6.4 Resíntesis por vocoder IA

  • Tras corregir el tono, el software utiliza el vocoder neuronal PC-NSF-HiFiGAN para resintetizar el audio
  • Preserva los formantes originales, la transposición de amplio rango no produce distorsión
  • Frecuencia de muestreo de salida: 44.1 kHz, calidad de sonido natural y nítida
  • Compatible con aceleración GPU (configurable en Preferencias)

6.5 Gestión multipista

  • Hasta 12 pistas independientes
  • Cada pista admite múltiples clips de audio
  • Operaciones de pista: Solo, Silencio, ajuste de volumen (perilla circular)
  • Medidores LED circulares muestran el volumen en tiempo real
  • Operaciones de clip: Arrastrar para mover, Dividir (Split), Fusionar (Merge), Eliminar

6.6 Deshacer/Rehacer

  • Guarda hasta 500 pasos de historial de operaciones
  • Compatible con todas las operaciones de edición: dibujo de notas, ajustes de parámetros, operaciones de clip, etc.
  • Atajos: Ctrl+Z (Deshacer), Ctrl+Y o Ctrl+Shift+Z (Rehacer)

7. Herramientas de edición

OpenTune ofrece cinco herramientas de edición profesionales, alternables desde la zona de herramientas del panel de parámetros o mediante atajos de teclado:

AutoTune — Herramienta de Corrección Automática (Atajo: 6)

Corrige automáticamente el tono a la nota objetivo más cercana según la escala actual. Ideal para corrección rápida por lotes.

Select — Herramienta de Selección (Atajo: 3)

Selecciona notas o regiones de tiempo. Tras la selección puede mover, eliminar, ajustar parámetros, etc.

Operaciones: Clic para seleccionar, arrastrar para selección por marco, Shift+clic para selección múltiple, Supr para eliminar.

DrawNote — Herramienta de Dibujo de Notas (Atajo: 2)

Dibuja bloques de notas manualmente. Arrastre en el rollo de piano para crear notas con tono y duración específicos.

Característica: Cada nota puede tener configuraciones independientes de Retune Speed, Vibrato Depth/Rate.

LineAnchor — Herramienta de Anclaje Lineal (Atajo: 4)

Coloca puntos de anclaje clave en la curva de tono; el software genera automáticamente transiciones de interpolación lineal entre anclajes. Ideal para controlar con precisión glissandos y transiciones de tono.

HandDraw — Herramienta de Dibujo Libre

Dibuja directamente la curva F0 para personalizar completamente el contorno tonal. Ideal para crear vibrato, glissando, pitch bend y otros efectos decorativos.

7.1 Audición de teclas del piano

Haga clic en las teclas del piano en el lado izquierdo del rollo de piano para reproducir en tiempo real una muestra de piano con el tono correspondiente, facilitando la confirmación del tono objetivo. Puede mantener presionado y arrastrar entre teclas para deslizarse entre notas; al soltar el ratón la reproducción se detiene automáticamente. La tecla pulsada se resalta en azul.

8. Parámetros detallados

8.1 Parámetros de corrección de tono

Retune Speed (Velocidad de corrección)
Rango: 0% – 100% | Valor predeterminado: 15%

Controla la velocidad e intensidad de la corrección de tono. Valores más altos = corrección más rápida y precisa; valores más bajos = más fluctuaciones originales preservadas.

  • Valores bajos (0–30%): Preserva el vibrato natural, ideal para sonido realista
  • Valores medios (30–70%): Equilibra precisión y naturalidad
  • Valores altos (70–100%): Corrección precisa, produce un efecto Auto-Tune notable

Por nota: Ajuste tras seleccionar notas; solo afecta las notas seleccionadas.

Vibrato Depth (Profundidad de vibrato)
Rango: 0 – 1 | Valor predeterminado: 0

Amplitud del vibrato artificial añadido a las notas. 0 = sin vibrato, 1 = profundidad máxima. Configurable por nota.

Vibrato Rate (Velocidad de vibrato)
Rango: 0 – 15 Hz | Valor predeterminado: 7.5 Hz

Controla la frecuencia de oscilación del vibrato artificial. El vibrato vocal natural suele estar en el rango de 5–8 Hz. Configurable por nota.

Note Split Threshold (Umbral de división de notas)
Rango: 0 – 100 cents | Valor predeterminado: 80 cents

Controla la sensibilidad de división en la generación automática de notas. Valores más bajos facilitan la división en notas independientes.

8.2 Parámetros de pista

Volume (Volumen)
Rango: 0 – 4.0x | Valor predeterminado: 1.0

Perilla de volumen de pista, admite ganancia superior a 1.0.

8.3 Parámetros de reproducción

BPM (Pulsaciones por minuto)
Rango: 20 – 300 BPM | Valor predeterminado: 120

Controla el tempo del proyecto y la cuadrícula de la línea de tiempo. Haga doble clic en el número para ingresar un valor preciso.

Tap Tempo

Toque repetidamente el botón Tap; el software calcula automáticamente el intervalo promedio para determinar el BPM. Registra hasta 5 toques.

13. Aceleración GPU

OpenTune es compatible con aceleración GPU para la inferencia del vocoder, lo que mejora significativamente la velocidad de renderizado.

13.1 Plataformas compatibles

  • Windows: Compatible con GPU NVIDIA / AMD / Intel mediante DirectML
  • macOS: Compatible con GPU de Apple mediante CoreML
  • Si no hay GPU dedicada o es incompatible, se revierte automáticamente a CPU sin afectar el uso

13.2 Configuración

En Preferencias → página Compartido → Prioridad de renderizado, seleccione:

  • Prioridad GPU (predeterminado): Usa automáticamente aceleración GPU
  • Prioridad CPU: Fuerza el uso de CPU, útil para problemas de compatibilidad GPU

14. Modo plugin VST3/ARA

El plugin OpenTune VST3 se integra profundamente con el DAW mediante el protocolo ARA2, permitiendo edición no destructiva.

14.1 Modo de funcionamiento

  • Se carga como extensión VST3 + ARA2 en DAW compatibles (como Studio One, Cubase, Logic Pro)
  • El DAW proporciona los datos de audio fuente mediante el protocolo ARA, sin necesidad de importar/exportar archivos
  • Los resultados de edición se reflejan en tiempo real en la reproducción de la línea de tiempo del DAW
  • Compatible con sincronización de transporte del host (Play/Stop/Seek) y seguimiento de BPM

16. Preguntas frecuentes

Q1: ¿No se muestra la curva de tono tras importar audio?

R: Asegúrese de que models/rmvpe.onnx existe y está completo. El audio debe contener contenido vocal melódico claro. Las pistas de solo acompañamiento/percusión no permiten extraer F0.

Q2: ¿La aceleración GPU no funciona?

R: Verifique lo siguiente:

  • Asegúrese de que DirectML.dll está en el directorio del programa
  • Confirme que los controladores GPU están actualizados a la última versión
  • En Preferencias, confirme que "Prioridad de renderizado" está en Prioridad GPU
  • Revise el archivo de registro para confirmar si el proveedor DML se cargó correctamente

Q3: ¿El consumo de memoria es muy alto?

R: v1.5 ya optimizó la gestión de memoria: el modelo F0 se libera automáticamente tras su uso. Si aún es alto, intente: desactivar la aceleración GPU (la sesión GPU consume memoria de video adicional), reducir el número de pistas abiertas simultáneamente.

Q4: ¿La calidad del audio corregido es anormal?

R: Posibles causas: archivo de modelo del vocoder dañado (vuelva a descargarlo), Retune Speed demasiado alto (pruebe por debajo del 30%), calidad deficiente de la grabación original.

Q5: ¿Cómo lograr un efecto de corrección de tono natural?

  • Use una Retune Speed baja (10–30%)
  • Conserve el vibrato original, no añada Vibrato artificial
  • Use la herramienta LineAnchor para ajustar con precisión las posiciones clave
  • Seleccione el modo "Prioridad Curva" para obtener un control más fino

Q6: ¿El plugin VST3 no carga en el DAW?

R: Asegúrese de que la carpeta OpenTune.vst3 está completa (incluye el subdirectorio Contents/x86_64-win/), y que los DLL de ONNX Runtime y los archivos de modelo están en la ubicación correcta. Vuelva a escanear el directorio VST3 en su DAW.

Q7: ¿Cómo cambiar el idioma de la interfaz?

R: Preferencias → Idioma (Language), seleccione el idioma deseado y la interfaz cambiará inmediatamente. Compatible con chino, inglés, japonés, ruso y español.
