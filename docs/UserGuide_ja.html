1. 製品概要

OpenTune は、ニューラルボコーダーベースのオープンソース AI ピッチ補正ソフトウェアです。従来の DSP アルゴリズムが直接ピッチシフトを行うのとは異なり、PC-NSF-HiFiGAN ボコーダーを使用して元のフォルマントを保持したまま声を再生成するため、極端なピッチ調整でも自然で安定した音質を実現します。

OpenTune は Standalone（スタンドアロン）と VST3/ARA（ホストプラグイン）の両方の動作モードを提供し、独立したマルチトラックエディターとしても、DAW に組み込んでリアルタイムの共同編集を行うことも可能です。

GPU アクセラレーションについて

OpenTune は DirectML（Windows）または CoreML（macOS）GPU アクセラレーションに対応しています。GPU が使用できない場合は自動的に CPU にフォールバックし、通常の使用に影響はありません。設定で手動で「CPU 優先」に切り替えることも可能です。

3. インストールと起動

3.1 ソフトウェアのダウンロード

GitHub Releases ページから最新バージョンをダウンロードしてください。

3.2 ファイル構成

  • OpenTune.exe — メインプログラム（Standalone）
  • OpenTune.vst3 — VST3 プラグイン（DAW 用）
  • onnxruntime.dll / onnxruntime_providers_shared.dll — ONNX Runtime
  • DirectML.dll — DirectML ランタイム（GPU アクセラレーション）
  • models/ — AI モデルフォルダ
    • rmvpe.onnx — ピッチ抽出モデル（~28MB）
    • pc_nsf_hifigan_44.1k_ONNX/ — ボコーダーモデル
  • docs/UserGuide.html — 本ユーザーガイド

3.3 プログラムの起動

OpenTune.exe をダブルクリックしてスタンドアロン版を起動します。初回起動時に AI モデルが自動的に初期化されます。VST3 プラグインは OpenTune.vst3 をホストソフトの VST3 スキャンディレクトリにコピーする必要があります。

上部ツールバーの左右両端にあるパネル切替ボタンをクリックすると、左側トラックパネルまたは右側パラメータパネルを非表示/表示し、「没入型メインキャンバス」モードにできます。

4.3 ビュー切替

  • トラックビュー (Track View)：アレンジビュー + ピアノロールを表示、複数オーディオクリップの管理に適しています
  • ピアノロールビュー (Piano Roll View)：全画面ピアノロール編集、精密なピッチ編集に集中

4.4 マウスホバーヒント

任意のコントロールにマウスを約0.6秒ホバーすると、ダークスタイルのフローティングヒントウィンドウが表示され、コントロールの機能説明とショートカットキー（存在する場合）が表示されます。ヒントテキストは画面言語に応じて自動的に切り替わります。

5. クイックスタート

1

オーディオをインポート

オーディオファイルをソフトウェアウィンドウにドラッグ＆ドロップするか、メニュー File → Import Audio からインポートします。WAV、MP3、FLAC、OGG、AIFF などの形式に対応しています。

2

自動ピッチ抽出

インポート後、ソフトウェアが自動的に元のピッチカーブを抽出し、ピアノロールに赤色のカーブとして表示します。

3

調性設定

ソフトウェアが自動的に調性を検出するか、手動でルート音（C〜B）と調性タイプ（長調、短調、半音階など8種類）を選択します。

4

ピッチ編集

自動補正ツール（ワンクリック補正）を使用するか、手動で音符/アンカー/手描きカーブを描画します。補正後のピッチは青色のカーブとして表示されます。

5

パラメータ調整

パラメータパネルで Retune Speed、Vibrato Depth/Rate などを調整します。音符ごとに独立したビブラートパラメータの設定をサポートします。

6

プレビューと書き出し

Space キーでプレビュー再生し、満足したら File → Export から書き出します。選択クリップ、単一トラック、またはバスミックスでの書き出しに対応しています。

6. 主要機能

6.1 オーディオインポート

  • 対応フォーマット：WAV、MP3、FLAC、OGG、AIFF など
  • インポート時に自動的に 44.1kHz にリサンプリング
  • ドラッグ＆ドロップインポート対応（ウィンドウに直接ドラッグ）
  • マルチトラック対応：最大 12 の独立したトラック、各トラックに複数クリップを配置可能
  • 複数ファイルの一括インポート対応（順次配置またはトラック分割）

6.2 ピッチ抽出（RMVPE）

  • インポート後、元のピッチカーブが自動抽出されます（赤色で表示）
  • RMVPE ディープラーニングモデルをベースに、高精度かつノイズ耐性に優れています
  • 50Hz – 1100Hz の周波数範囲でボーカル検出に対応

6.3 調性検出と選択

Krumhansl-Schmuckler アルゴリズムによる自動調性検出を内蔵し、8種類の調性タイプに対応：

調性略称説明
長調Maj.自然長音階
短調Min.自然短音階
半音階Chr.全12半音が使用可能（調性フィルターなし）
和声短調H.Min.第7音を半音上げた短調
ドリアンDor.ジャズ/ブルースでよく使用
ミクソリディアンMix.第7音を半音下げた長調

短調五声m.Pent.ブルース五音音階

6.4 AI ボコーダーリシンセシス

  • ピッチ補正後、PC-NSF-HiFiGAN ニューラルボコーダーでオーディオを再合成します
  • 元のフォルマントを保持し、広範囲のピッチシフトでも歪みなし
  • 出力サンプルレート 44.1kHz、自然でクリアな音質
  • GPU アクセラレーションに対応（設定で構成可能）

6.5 マルチトラック管理

  • 最大 12 の独立したトラック
  • 各トラックに複数のオーディオクリップを配置可能
  • トラック操作：Solo、Mute、ボリューム調整（円形ノブ）
  • 円形 LED レベルメーターでリアルタイム音量表示
  • クリップ操作：ドラッグ移動、分割（Split）、結合（Merge）、削除

6.6 取り消し/やり直し

  • 最大 500 ステップの操作履歴を保存
  • すべての編集操作に対応：音符描画、パラメータ調整、クリップ操作など
  • ショートカット：Ctrl+Z（取り消し）、Ctrl+Y または Ctrl+Shift+Z（やり直し）

7. 編集ツール

OpenTune は5つのプロフェッショナル編集ツールを提供し、パラメータパネルのツールエリアまたはショートカットキーで切り替えます：

AutoTune — 自動補正ツール (ショートカット: 6)

現在の調性に基づいてピッチを最も近い目標音符に自動補正します。迅速な一括補正に最適です。

Select — 選択ツール (ショートカット: 3)

音符または時間領域を選択します。選択後、移動、削除、パラメータ調整などの操作が可能です。

操作：クリック選択、ドラッグボックス選択、Shift+クリック複数選択、Delete 削除。

DrawNote — 音符描画ツール (ショートカット: 2)

手動で音符ブロックを描画します。ピアノロール内でドラッグして音符を作成し、ピッチと長さを指定します。

特徴：各音符に Retune Speed、Vibrato Depth/Rate パラメータを個別に設定可能です。

LineAnchor — ラインアンカーツール (ショートカット: 4)

ピッチカーブ上にキーアンカーを配置し、ソフトウェアがアンカー間を自動的に直線補間で遷移させます。ポルタメントや転音の精密な制御に最適です。

HandDraw — 手描きツール

F0 カーブを直接手描きし、ピッチの動きを完全にカスタマイズします。ビブラート、ポルタメント、ベンドなどの装飾効果の描画に適しています。

7.1 鍵盤試聴

ピアノロール左側の鍵盤エリアをクリックすると、対応するピッチのピアノサンプルがリアルタイムで再生され、目標ピッチの確認に役立ちます。押したままドラッグすると異なるキー間をスライドで切り替え、マウスを離すと自動的に再生が停止します。押されたキーは青色でハイライト表示されます。

8. パラメータ詳細

8.1 ピッチ補正パラメータ

Retune Speed（補正速度）
範囲：0% – 100% | デフォルト：15%

ピッチ補正の速度と強度を制御します。値が大きいほど補正が速く精密になり、小さいほど元のピッチの揺れをより多く保持します。

  • 低値（0–30%）：自然なビブラートを保持、リアルな質感を追求
  • 中値（30–70%）：精度と自然さのバランス
  • 高値（70–100%）：精密な補正、顕著な Auto-Tune 効果を生成

音符単位：音符を選択して調整すると、選択部分のみに影響します。

Vibrato Depth（ビブラート深度）
範囲：0 – 1 | デフォルト：0

音符に人工ビブラートの振幅を追加します。0 = 追加なし、1 = 最大深度。音符ごとの設定に対応。

Vibrato Rate（ビブラート速度）
範囲：0 – 15 Hz | デフォルト：7.5 Hz

人工ビブラートの振動周波数を制御します。人間の自然なビブラートは通常 5–8 Hz の範囲です。音符ごとの設定に対応。

Note Split Threshold（音符分割閾値）
範囲：0 – 100 セント | デフォルト：80 セント

自動音符生成時の分割感度を制御します。値が小さいほど、独立した音符に分割されやすくなります。

8.2 トラックパラメータ

Volume（音量）
範囲：0 – 4.0 倍 | デフォルト：1.0

トラック音量ノブ、1.0 を超えるゲインブーストに対応。

8.3 再生パラメータ

BPM（Beats Per Minute）
範囲：20 – 300 BPM | デフォルト：120

プロジェクトのテンポとタイムライングリッドを制御します。数値をダブルクリックして正確な値を直接入力できます。

Tap Tempo

Tap ボタンを連続クリックすると、ソフトウェアが平均間隔から自動的に BPM を計算します。最大5回のクリックを記録します。

9. 編集モード

OpenTune は2つの編集モードを提供し、ツールの動作を決定します。設定 → 編集モードから切り替え可能です。

9.1 カーブ優先編集（デフォルト）

  • LineAnchor セグメントを選択し、独立した Retune Speed を設定可能
  • パラメータパネルは選択中の LineAnchor セグメントのパラメータを優先表示
  • AutoTune は選択中の時間領域を優先して作用

9.2 音符優先編集

  • HandDraw / LineAnchor 描画後、影響を受ける音符を自動選択
  • AutoTune は選択中の音符範囲を優先して作用
  • パラメータパネルは選択中の音符のパラメータを優先表示

13. GPU アクセラレーション

OpenTune は GPU アクセラレーションによるボコーダー推論の高速化に対応し、レンダリング速度を大幅に向上できます。

13.1 対応プラットフォーム

  • Windows：DirectML 経由で NVIDIA / AMD / Intel GPU に対応
  • macOS：CoreML 経由で Apple GPU に対応
  • 独立 GPU がない、または GPU 非互換の場合は自動的に CPU にフォールバックし、使用に影響はありません

13.2 設定方法

設定 → 共有ページ → レンダリング優先で選択：

  • GPU 優先（デフォルト）：自動的に GPU アクセラレーションを使用
  • CPU 優先：CPU を強制使用、GPU 互換性の問題がある場合に使用

14. VST3/ARA プラグインモード

OpenTune VST3 プラグインは ARA2 プロトコルを通じて DAW と深く統合され、非破壊編集を実現します。

14.1 動作方式

  • VST3 + ARA2 拡張として互換 DAW（Studio One、Cubase、Logic Pro など）にロード
  • DAW が ARA プロトコルを通じてオーディオソースデータを提供、ファイルのインポート/エクスポート不要
  • 編集結果は DAW のタイムライン再生にリアルタイム反映
  • ホストトランスポート同期（Play/Stop/Seek）および BPM 追従に対応

16. よくある質問

Q1: オーディオをインポートしたがピッチカーブが表示されない？

A: models/rmvpe.onnx が存在し完全であることを確認してください。オーディオには明瞭なボーカルメロディーが含まれている必要があります。純粋な伴奏や打楽器のみでは F0 を抽出できません。

Q2: GPU アクセラレーションが効いていない？

A: 以下の点を確認してください：

  • DirectML.dll がプログラムディレクトリに存在すること
  • GPU ドライバが最新バージョンに更新されていること
  • 設定で「レンダリング優先」が GPU 優先に設定されていること
  • ログファイルで DML プロバイダーが正常にロードされたか確認

Q3: メモリ使用量が多い？

A: v1.5 ではメモリ管理が最適化されており、F0 モデルは使用後に自動解放されます。それでも高い場合は、GPU アクセラレーションを無効にする（GPU セッションは追加のビデオメモリを消費します）、同時に開くトラック数を減らすなどをお試しください。

Q4: 補正後の音質に異常がある？

A: 考えられる原因：ボコーダーモデルファイルの破損（再ダウンロード）、Retune Speed の設定が高すぎる（30% 以下に下げてお試しください）、元の録音品質が低い。

Q5: 自然なピッチ補正効果を得るには？

  • 低めの Retune Speed（10–30%）を使用する
  • 元のビブラートを保持し、人工 Vibrato を追加しない
  • LineAnchor ツールでキーポイントを精密に調整する
  • 「カーブ優先編集」モードを選択してより精密な制御を得る

Q6: VST3 プラグインが DAW でロードできない？

A: OpenTune.vst3 フォルダが完全であること（Contents/x86_64-win/ サブディレクトリを含む）、ONNX ランタイム DLL とモデルファイルが正しい位置にあることを確認してください。DAW で VST3 ディレクトリを再スキャンしてください。

Q7: 画面言語を切り替えるには？

A: 設定 → 言語（Language）で目的の言語を選択すると、画面が即座に切り替わります。中文、English、日本語、Русский、Español に対応しています。
