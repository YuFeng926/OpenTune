1. Introduction

OpenTune is an open-source AI pitch correction application powered by a neural vocoder. Unlike traditional DSP algorithms that transpose by directly shifting pitch, OpenTune uses the PC-NSF-HiFiGAN vocoder to regenerate vocals while preserving the original formants. Even extreme pitch adjustments sound natural, solid, and artifact-free.

OpenTune offers both Standalone and VST3/ARA (DAW plugin) modes — use it as an independent multi-track editor, or embed it into your DAW for collaborative real-time editing.

3. Installation & Launch

3.1 Download

Download the latest version from the GitHub Releases page.

3.2 File Structure

  • OpenTune.exe — Main program (Standalone)
  • OpenTune.vst3 — VST3 plugin (for DAW use)
  • onnxruntime.dll / onnxruntime_providers_shared.dll — ONNX Runtime
  • DirectML.dll — DirectML Runtime (GPU acceleration)
  • models/ — AI model folder
    • rmvpe.onnx — Pitch extraction model (~28MB)
    • pc_nsf_hifigan_44.1k_ONNX/ — Vocoder model
  • docs/UserGuide.html — This user guide

3.3 Launching

Double-click OpenTune.exe to start the standalone version. AI models are automatically initialized on first launch. For VST3 plugin use, copy the OpenTune.vst3 folder to your DAW's VST3 scan directory.

5. Quick Start

1

Import Audio

Drag and drop audio files into the application window, or use File → Import Audio. Supported formats: WAV, MP3, FLAC, OGG, AIFF, and more.

2

Automatic Pitch Extraction

After import, the software automatically extracts the original pitch curve, displayed as a red curve in the piano roll.

3

Scale Settings

The software automatically detects the scale, or you can manually select the root note (C–B) and scale type (Major, Minor, Chromatic, etc. — 8 types).

4

Edit Pitch

Use the AutoTune tool for one-click correction, or manually draw notes/anchors/freehand curves. Corrected pitch is displayed as a blue curve.

5

Adjust Parameters

Adjust Retune Speed, Vibrato Depth/Rate, etc. in the parameter panel. Per-note vibrato settings are supported.

6

Preview & Export

Press Space to play and preview. When satisfied, export via File → Export. Supports exporting selected clips, individual tracks, or master bus mix.

6. Core Features

6.1 Audio Import

  • Supported formats: WAV, MP3, FLAC, OGG, AIFF, and more
  • Auto-resampled to 44.1kHz on import
  • Drag-and-drop import (drop directly into the window)
  • Multi-track support: up to 12 independent tracks, each capable of holding multiple clips
  • Batch import of multiple files (sequential layout or split by track)

6.2 Pitch Extraction (RMVPE)

  • Based on the RMVPE deep-learning model — high accuracy and strong noise immunity
  • Supports vocal detection in the 50Hz – 1100Hz frequency range

  • Preserves original formants — wide transposition ranges sound natural

  • GPU acceleration supported (configurable in Preferences)

7. Editing Tools

OpenTune provides five professional editing tools, switchable via the tool area in the parameter panel or by keyboard shortcut:

AutoTune — Automatic Correction Tool (Shortcut: 6)

Automatically corrects pitch to the nearest target note based on the current scale. Ideal for quick batch correction.

Select — Selection Tool (Shortcut: 3)

Select notes or time regions. After selection, you can move, delete, adjust parameters, etc.

Operations: Click to select, drag to marquee-select, Shift+click to multi-select, Delete to remove.

DrawNote — Note Drawing Tool (Shortcut: 2)

Manually draw note blocks. Drag in the piano roll to create notes with specified pitch and duration.

Feature: Each note can have independent Retune Speed, Vibrato Depth/Rate settings.

LineAnchor — Line Anchor Tool (Shortcut: 4)

Place key anchor points on the pitch curve; the software automatically generates linear interpolation transitions between anchors. Ideal for precise control of glides and pitch bends.

HandDraw — Freehand Tool

Draw the F0 curve directly by hand for fully customized pitch shaping. Ideal for drawing vibrato, glides, bends, and other ornamental effects.

7.1 Piano Key Audition

Click the piano keys on the left side of the piano roll to play the corresponding pitch as a piano sample in real time, making it easy to confirm target pitches. You can hold and drag across keys to glide between notes; releasing the mouse stops playback automatically. The pressed key is highlighted in blue.

8. Parameter Reference

8.1 Pitch Correction Parameters

Retune Speed
Range: 0% – 100% | Default: 15%

Controls the speed and intensity of pitch correction. Higher values yield faster, more precise correction; lower values preserve more of the original pitch fluctuation.

  • Low (0–30%): Preserves natural vibrato — ideal for a realistic feel
  • Medium (30–70%): Balances precision and naturalness
  • High (70–100%): Precise correction — produces the distinct Auto-Tune effect

Per-note: Select a note before adjusting to affect only that note.

Vibrato Depth
Range: 0 – 1 | Default: 0

Adds artificial vibrato amplitude to notes. 0 = no addition, 1 = maximum depth. Supports per-note setting.

Vibrato Rate
Range: 0 – 15 Hz | Default: 7.5 Hz

Controls the oscillation frequency of artificial vibrato. Natural vocal vibrato typically falls in the 5–8 Hz range. Supports per-note setting.

Note Split Threshold
Range: 0 – 100 cents | Default: 80 cents

Controls the split sensitivity during automatic note generation. Lower values make it easier to split into independent notes.

8.2 Track Parameters

Volume
Range: 0 – 4.0× | Default: 1.0

Track volume knob, supports gain amplification beyond 1.0.

8.3 Playback Parameters

BPM (Beats Per Minute)
Range: 20 – 300 BPM | Default: 120

Controls the project tempo and timeline grid. Double-click the number to enter an exact value directly.

Tap Tempo

Continuously tap the Tap button; the software automatically calculates the average interval to derive BPM. Up to 5 taps are recorded.

9. Editing Schemes

OpenTune provides two editing schemes that determine tool behavior. Switch via Preferences → Editing Scheme.

9.1 Curve-First Editing (Default)

  • LineAnchor segments can be selected and given independent Retune Speed
  • The parameter panel prioritizes displaying parameters of the selected LineAnchor segment
  • AutoTune prioritizes acting on the selected time region

9.2 Note-First Editing

  • After HandDraw / LineAnchor drawing, affected notes are automatically selected
  • AutoTune prioritizes acting on the selected note range
  • The parameter panel prioritizes displaying parameters of the selected note

13. GPU Acceleration

OpenTune supports GPU-accelerated vocoder inference, significantly improving rendering speed.

13.1 Supported Platforms

  • Windows: Supports NVIDIA / AMD / Intel GPUs via DirectML
  • macOS: Supports Apple GPU via CoreML
  • Automatically falls back to CPU if no discrete GPU is present or GPU is incompatible — normal usage unaffected

13.2 Configuration

In Preferences → Shared → Render Priority, select:

  • GPU Preferred (default): Automatically use GPU acceleration
  • CPU Preferred: Force CPU usage — suitable when GPU compatibility issues arise

14. VST3/ARA Plugin Mode

The OpenTune VST3 plugin integrates deeply with DAWs via the ARA2 protocol, enabling non-destructive editing.

14.1 How It Works

  • Loaded as a VST3 + ARA2 extension in compatible DAWs (e.g., Studio One, Cubase, Logic Pro)
  • The DAW provides audio source data via the ARA protocol — no file import/export needed
  • Editing results are reflected in real time during DAW timeline playback
  • Supports host transport sync (Play/Stop/Seek) and BPM following

16. FAQ

Q1: No pitch curve after importing audio?

A: Ensure models/rmvpe.onnx exists and is intact. The audio must contain clear vocal melodic content. Pure accompaniment/percussion cannot extract F0.

Q2: GPU acceleration not working?

A: Check the following:

  • Ensure DirectML.dll is in the program directory
  • Confirm GPU drivers are updated to the latest version
  • Verify "Render Priority" is set to GPU Preferred in Preferences
  • Check log files to confirm the DML provider loaded successfully

Q3: High memory usage?

A: v1.5 has optimized memory management — the F0 model is automatically released after use. If usage remains high, try: disabling GPU acceleration (GPU sessions consume additional memory), reducing the number of simultaneously open tracks.

Q4: Abnormal audio quality after correction?

A: Possible causes: corrupted vocoder model files (re-download), Retune Speed set too high (try lowering below 30%), poor original recording quality.

Q5: How to achieve a natural tuning effect?

  • Use a lower Retune Speed (10–30%)
  • Preserve original vibrato; do not add artificial Vibrato
  • Use the LineAnchor tool for fine adjustments at key points
  • Choose "Curve-First Editing" scheme for more refined control

Q6: VST3 plugin won't load in DAW?

A: Ensure the OpenTune.vst3 folder is complete (contains Contents/x86_64-win/ subdirectory), and the ONNX Runtime DLLs and model files are in the correct locations. Re-scan the VST3 directory in your DAW.

Q7: How to switch the interface language?

A: Preferences → Language — select your target language and the interface updates immediately. Supports Chinese, English, Japanese, Russian, and Spanish.
