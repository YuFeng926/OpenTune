1. 产品简介

OpenTune 是一款基于神经声码器的开源 AI 修音软件。与传统 DSP 算法直接移调不同，它使用 NSF-HiFiGAN 声码器在保留原始共振峰的前提下重新生成人声，即使进行极端音高调整，也能保持自然、扎实的音质。

核心优势

特性 说明
共振峰不变 改变音高时不影响声音的底色，告别"鸭子叫"或"换人唱"的失真感
大范围移调 支持夸张的音高修正与转调，音质依然清晰稳定
AI 重合成 基于深度学习的声码器技术，而非传统移调算法
类 Auto-Tune 工作流 手绘、音符、锚点工具一应俱全，快速进入心流状态
自动调式检测 内置类 Auto-Key 功能，自动检测调式（大调/小调）
开源免费 永久免费，社区驱动，持续迭代
💡 技术亮点
OpenTune 使用 RMVPE 进行高精度音高提取，配合 PC-NSF-HiFiGAN 神经声码器实现高质量的重合成。音频统一采用 44.1kHz 采样率处理。

项目 要求
处理器 Intel Core i7 或更高
内存 16 GB RAM 或更高

✓ GPU 加速
OpenTune 支持 NVIDIA GPU 的 CUDA/DirectML 加速。如果检测到兼容显卡，AI 推理将自动使用 GPU，大幅提升处理速度。无独立显卡时自动回退到 CPU 模式运行。

3. 安装与启动

3.1 下载软件

前往 GitHub Releases 页面下载最新 Windows 独立运行包。

3.2 解压文件

将下载的压缩包解压到任意目录，确保以下文件完整：

  • OpenTune.exe - 主程序
  • onnxruntime.dll - ONNX Runtime 运行库
  • models/ - AI 模型文件夹
    • rmvpe.onnx - 音高提取模型
    • pc_nsf_hifigan_44.1k_ONNX/ - 声码器模型

3.3 启动程序

双击 OpenTune.exe 即可启动软件。首次启动时，软件会自动初始化 AI 模型，可能需要等待几秒钟。

✓ 模型加载
确保模型文件与主程序在同一目录下，否则软件无法正常工作。模型文件总大小约 60MB。

5. 快速入门

以下是一个典型的工作流程，帮助你快速上手 OpenTune：

1

导入音频

将音频文件拖放到软件窗口，或通过菜单 File → Import Audio 导入。支持 WAV、MP3、FLAC 等格式。

2

自动音高提取

导入后软件自动运行 RMVPE 提取原始 F0 曲线，在钢琴卷帘视图中显示红色曲线。

3

设置调式

软件自动检测调式，或手动在传输控制栏选择主音（C, C#, D...）和调式（Major/Minor）。

4

编辑音高曲线

使用自动修正工具或手动绘制音符、锚点，调整音高曲线。蓝色曲线显示修正后的音高。

5

调整参数

在参数面板调节 Retune Speed、Vibrato 等参数，实时预览效果。

6

预览与导出

点击播放按钮预览效果，满意后通过 File → Export 导出处理后的音频。

💡 快捷操作
导入音频后可以直接使用 AutoTune 工具进行一键修正，适合快速处理。精细调整时再使用其他工具。

6. 核心功能

6.1 音频导入

OpenTune 支持多种音频格式的导入：

  • 支持格式：WAV、MP3、FLAC、OGG、AIFF 等
  • 导入时自动重采样至 44.1kHz
  • 支持拖放导入
  • 多轨道支持：最多 12 个独立轨道

6.2 音高提取

软件使用 RMVPE（Robust Multi-Scale Pitch Estimation）进行高精度音高提取：

  • 帧率：100fps（每帧 10ms）
  • 频率范围：默认 50Hz - 1100Hz（可调整）
  • 特性：支持八度错误修正、静音段检测、间隙填充

6.3 调式检测

内置基于 Krumhansl-Schmuckler 算法的自动调式检测：

  • 支持 12 种主音：C, C#, D, D#, E, F, F#, G, G#, A, A#, B
  • 支持调式：大调（Major）、小调（Minor）、半音阶
  • 提供置信度评估

6.4 音高校正

校正模式

模式 说明
自动修正 根据调式自动将音高校正到最接近的目标音符
音符编辑 手动绘制音符块，指定每个音符的基准音高
手绘曲线 直接绘制 F0 曲线，实现完全自定义的音高走向
锚点工具 设置关键锚点，自动生成平滑过渡的音高线

音高曲线显示

原始 F0 曲线（红色）
修正后 F0 曲线（蓝色）

6.5 AI 重合成

修正后的音高曲线通过神经声码器重合成音频：

  • 声码器：PC-NSF-HiFiGAN（Pitch Controllable NSF HiFiGAN）
  • 采样率：44.1kHz
  • 特点：保留原始共振峰，音质自然
  • 缓存机制：分块渲染并缓存，避免重复计算

6.6 多轨道管理

OpenTune 支持多轨道工作流程：

  • 最多 12 个独立轨道
  • 每个轨道支持多个音频片段（Clip）
  • 轨道操作：Solo、Mute、音量调节
  • 片段操作：移动、分割、合并、删除
  • 环形 LED 电平表实时显示音量

6.7 撤销/重做

支持完整的撤销/重做历史：

  • 最多保存 500 步操作历史
  • 支持所有编辑操作（音符绘制、参数调整等）
  • 快捷键：Ctrl+Z（撤销）、Ctrl+Y 或 Ctrl+Shift+Z（重做）

6.8 快捷键系统

内置可自定义的快捷键系统：

  • 支持常用操作的快捷键绑定
  • 可在设置中自定义快捷键
  • 支持冲突检测和提示
  • 提供快捷键重置功能

6.9 GPU 加速

支持 GPU 加速 AI 推理：

  • 自动检测 NVIDIA GPU 并启用 CUDA/DirectML 加速
  • CPU 回退模式确保兼容性
  • 显著提升处理速度

6.10 多主题系统

提供多种视觉主题选择：

  • BlueBreeze - 浅色主题，简约现代
  • DarkBlueGrey - 深色主题，专业录音室风格
  • Aurora - 深色霓虹主题，科技感十足
  • 通过菜单 View → Theme 切换主题

7. 编辑工具

OpenTune 提供五种专业编辑工具，通过参数面板的工具选择区切换：

🎹 AutoTune - 自动修正工具

根据当前调式自动将音高校正到最接近的目标音符。适合快速处理、批量修正。

使用场景：当音频基本在调上，只需要微调时使用。

🖱️ Select - 选择工具

选择音符或时间区域。选中后可进行移动、删除、参数调整等操作。

操作：单击选择、拖动框选、Delete 删除选中内容。

✏️ DrawNote - 绘制音符工具

手动绘制音符块。在钢琴卷帘中拖动创建音符，指定音高和时长。

特点：每个音符可独立设置参数（Retune Speed、颤音等）。

📍 LineAnchor - 线锚点工具

在音高曲线上设置关键锚点，软件自动在锚点之间生成平滑过渡。

优势：精确控制关键位置，适合处理滑音、转音。

🖌️ HandDraw - 手绘工具

直接手绘 F0 曲线，完全自定义音高走向。适合精细的音高装饰。

技巧：可绘制颤音、滑音、弯音等细节效果。

工具切换

在参数面板点击对应的工具按钮，或使用快捷键（见第 9 章）。

8. 参数详解

8.1 音高校正参数

Retune Speed（校正速度）
范围：0% - 100% | 默认值：15%

控制音高校正的速度。值越大，校正越快越精确；值越小，保留更多原始音高波动，效果更自然。

  • 低值（0-30%）：保留自然颤音，适合追求真实感的处理
  • 中值（30-70%）：平衡精度与自然度
  • 高值（70-100%）：精确校正，产生明显的 Auto-Tune 效果
Vibrato Depth（颤音深度）
范围：0 - 1 | 默认值：0

为音符添加人工颤音的深度。0 表示不添加，1 表示最大深度。

Vibrato Rate（颤音速率）
范围：0 - 15 Hz | 默认值：7.5 Hz

控制人工颤音的振动频率。人声自然颤音通常在 5-8 Hz 范围。

Note Split Threshold（音符分割阈值）
范围：0 - 100 音分 | 默认值：80 音分

控制自动音符分割的灵敏度。值越小，越容易分割成独立音符；值越大，倾向于合并相邻音高。

8.2 轨道参数

Volume（音量）
范围：0 - 4.0 倍 | 默认值：1.0

轨道音量控制。支持超过 1.0 的增益。

Solo / Mute

轨道独奏/静音开关。Solo 时其他轨道静音，Mute 时该轨道静音。

8.3 播放参数

BPM（每分钟节拍数）
范围：20 - 300 BPM | 默认值：120

项目速度。影响时间轴网格和节拍标记。

Tap Tempo

点击 Tap 按钮多次，软件自动计算平均 BPM。最多记录 5 次点击间隔。

10. 导出音频

OpenTune 支持多种导出选项：

10.1 导出格式

  • 格式：WAV
  • 采样率：44.1kHz
  • 位深度：32-bit Float
  • 声道：单声道（Clip/Track）/ 立体声（Master Mix）

  4. 等待渲染完成
💡 提示
导出时会自动触发最终渲染，确保所有修改都已应用到输出音频中。

11. 常见问题

Q1: 导入音频后没有显示音高曲线？

A: 请确保 models/rmvpe.onnx 模型文件存在且完整。首次加载模型可能需要几秒钟。如果问题持续，请检查音频是否包含明显的人声内容。

Q2: 音频处理速度很慢？

A: 当前版本主要依赖 CPU 进行 AI 推理。建议：

  • 关闭其他占用 CPU 的程序
  • 使用较短的分段进行处理
  • 升级到更高性能的 CPU

Q3: 修正后的音质有异常？

A: 可能原因：

  • 声码器模型损坏 - 重新下载模型文件
  • 原始音频质量较差 - 尝试调整 Note Split 阈值
  • 参数设置不当 - 降低 Retune Speed 值

Q4: 如何实现自然的音高修正？

A: 建议：

  • 使用较低的 Retune Speed（10-30%）
  • 保留原始颤音，不添加人工颤音
  • 使用 Line Anchor 工具进行精细调整，而非全自动修正

Q5: 支持实时处理吗？

A: 当前版本不支持实时处理。需要先导入音频、编辑后导出。后续版本将支持 VST3 插件和 ARA2，实现 DAW 内实时处理。

Q6: 可以在 DAW 中使用吗？

A: 当前仅支持 Windows Standalone 独立运行。VST3 插件版本即将推出，届时可在宿主软件中直接使用。

Q7: 如何报告 Bug 或提出功能建议？

A: 请通过 GitHub Issues 页面提交问题或建议，我们会积极跟进改进。
