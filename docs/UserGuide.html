1. 产品简介

OpenTune 是一款基于神经声码器的开源 AI 修音软件。与传统 DSP 算法直接移调不同，它使用 PC-NSF-HiFiGAN 声码器在保留原始共振峰的前提下重新生成人声，即使进行极端音高调整，也能保持自然、扎实的音质。

OpenTune 同时提供 Standalone（独立运行）和 VST3/ARA（宿主插件）两种工作模式，可作为独立的多轨编辑器使用，也可嵌入 DAW 中进行实时协作编辑。

OpenTune 支持 DirectML（Windows）或 CoreML（macOS）GPU 加速。若 GPU 不可用会自动回退 CPU，不影响正常使用。可在偏好设置中手动切换为"CPU 优先"。

3. 安装与启动

3.1 下载软件

前往 GitHub Releases 页面下载最新版本。

3.2 文件结构

  • OpenTune.exe — 主程序（Standalone）
  • OpenTune.vst3 — VST3 插件（用于 DAW）
  • onnxruntime.dll / onnxruntime_providers_shared.dll — ONNX Runtime
  • DirectML.dll — DirectML 运行时（GPU 加速）
  • models/ — AI 模型文件夹
    • rmvpe.onnx — 音高提取模型（~28MB）
    • pc_nsf_hifigan_44.1k_ONNX/ — 声码器模型
  • docs/UserGuide.html — 本用户指南

3.3 启动程序

双击 OpenTune.exe 即可启动独立版。首次启动时自动初始化 AI 模型。VST3 插件需将 OpenTune.vst3 复制到宿主软件的 VST3 扫描目录。

调式设置

软件自动检测调式，或手动选择根音（C~B）和调式类型（大调、小调、半音阶等 8 种）。

编辑音高

使用自动修正工具（一键校正）或手动绘制音符/锚点/手绘曲线。修正后的音高显示为蓝色曲线。

调整参数

在参数面板调节 Retune Speed、Vibrato Depth/Rate 等。支持逐音符设置独立颤音参数。

6

预览与导出

按 Space 播放预览，满意后通过 File → Export 导出。支持导出选中片段、单轨道或总线混音。

6. 核心功能

6.1 音频导入

  • 支持格式：WAV、MP3、FLAC、OGG、AIFF 等
  • 导入时自动重采样至 44.1kHz
  • 支持拖放导入（直接拖入窗口）
  • 多轨道支持：最多 12 个独立轨道，每轨道可放置多个片段
  • 支持批量导入多个文件（顺序排列或分轨）

6.2 音高提取（RMVPE）

  • 导入音频后自动提取原始音高曲线（红色显示）
  • 基于 RMVPE 深度学习模型，精度高、噪声抗性强
  • 支持 50Hz – 1100Hz 频率范围的人声检测

6.3 调式检测与选择

内置 Krumhansl-Schmuckler 算法的自动调式检测，支持 8 种调式类型：

调式缩写说明
大调Maj.自然大调音阶
小调Min.自然小调音阶
半音阶Chr.所有 12 个半音均可用（无调式过滤）
和声小调H.Min.升七级的小调
多利亚Dor.爵士/布鲁斯常用
混合利底亚Mix.降七级的大调
大调五声Pent.五声音阶（中国风）
小调五声m.Pent.布鲁斯五声音阶

6.4 AI 声码器重合成

  • 修正音高后，软件使用 PC-NSF-HiFiGAN 神经声码器重新合成音频
  • 保留原始共振峰，大范围移调不失真
  • 输出采样率 44.1kHz，音质自然清晰
  • 支持 GPU 加速（可在偏好设置中配置）

6.5 多轨道管理

  • 最多 12 个独立轨道
  • 每轨道支持多个音频片段
  • 轨道操作：Solo、Mute、音量调节（圆形旋钮）
  • 圆形 LED 电平表实时显示音量
  • 片段操作：拖动移动、分割（Split）、合并（Merge）、删除

6.6 撤销/重做

  • 最多保存 500 步操作历史
  • 支持所有编辑操作：音符绘制、参数调整、片段操作等
  • 快捷键：Ctrl+Z（撤销）、Ctrl+Y 或 Ctrl+Shift+Z（重做）

7. 编辑工具

OpenTune 提供五种专业编辑工具，通过参数面板的工具区或快捷键切换：

AutoTune — 自动修正工具 (快捷键: 6)

根据当前调式自动将音高校正到最接近的目标音符。适合快速批量修正。

Select — 选择工具 (快捷键: 3)

选择音符或时间区域。选中后可进行移动、删除、参数调整等操作。

操作：单击选择、拖动框选、Shift+点击多选、Delete 删除。

DrawNote — 绘制音符工具 (快捷键: 2)

手动绘制音符块。在钢琴卷帘中拖动创建音符，指定音高和时长。

特点：每个音符可独立设置 Retune Speed、Vibrato Depth/Rate 参数。

LineAnchor — 线锚点工具 (快捷键: 4)

在音高曲线上放置关键锚点，软件在锚点之间自动生成直线插值过渡。适合精确控制滑音、转音。

HandDraw — 手绘工具

直接手绘 F0 曲线，完全自定义音高走向。适合绘制颤音、滑音、弯音等装饰效果。

7.1 钢琴键试听

点击钢琴卷帘左侧的钢琴键区域可实时播放对应音高的钢琴采样音频，方便确认目标音高。支持按住拖拽在不同琴键之间滑动切换，松开鼠标后自动停止播放。被按下的琴键会以蓝色高亮显示。

8. 参数详解

8.1 音高校正参数

Retune Speed（校正速度）
范围：0% – 100% | 默认值：15%

控制音高校正的速度和强度。值越大，校正越快越精确；值越小，保留更多原始音高波动。

  • 低值 (0–30%)：保留自然颤音，适合追求真实感
  • 中值 (30–70%)：平衡精度与自然度
  • 高值 (70–100%)：精确校正，产生明显的 Auto-Tune 效果

逐音符：选中音符后调整，只影响选中音符。

Vibrato Depth（颤音深度）
范围：0 – 1 | 默认值：0

为音符添加人工颤音的振幅。0 = 不添加，1 = 最大深度。支持逐音符设置。

Vibrato Rate（颤音速率）
范围：0 – 15 Hz | 默认值：7.5 Hz

控制人工颤音的振动频率。人声自然颤音通常在 5–8 Hz 范围。支持逐音符设置。

Note Split Threshold（音符分割阈值）
范围：0 – 100 音分 | 默认值：80 音分

控制自动音符生成时的分割灵敏度。值越小，越容易分割成独立音符。

8.2 轨道参数

Volume（音量）
范围：0 – 4.0 倍 | 默认值：1.0

轨道音量旋钮，支持超过 1.0 的增益放大。

8.3 播放参数

BPM（每分钟节拍数）
范围：20 – 300 BPM | 默认值：120

控制项目速度和时间轴网格。双击数字可直接输入精确值。

Tap Tempo

连续点击 Tap 按钮，软件自动计算平均间隔得出 BPM。最多记录 5 次点击。

9. 编辑方案

13. GPU 加速

OpenTune 支持 GPU 加速声码器推理，可显著提升渲染速度。

13.1 支持的平台

  • Windows：通过 DirectML 支持 NVIDIA / AMD / Intel GPU
  • macOS：通过 CoreML 支持 Apple GPU
  • 无独立显卡或 GPU 不兼容时自动回退 CPU，不影响使用

13.2 设置方法

在偏好设置 → 共享页面 → 渲染优先级中选择：

  • GPU 优先（默认）：自动使用 GPU 加速
  • CPU 优先：强制使用 CPU，适合 GPU 兼容性问题时使用

14. VST3/ARA 插件模式

OpenTune VST3 插件通过 ARA2 协议与 DAW 深度集成，实现非破坏性编辑。

14.1 工作方式

  • 以 VST3 + ARA2 扩展加载到兼容 DAW（如 Studio One、Cubase、Logic Pro）
  • DAW 通过 ARA 协议提供音频源数据，无需导入/导出文件
  • 编辑结果实时反映到 DAW 时间线播放中

16. 常见问题

Q1: 导入音频后没有显示音高曲线？

A: 请确保 models/rmvpe.onnx 存在且完整。音频需包含明显的人声旋律内容。纯伴奏/打击乐无法提取 F0。

Q2: GPU 加速没有生效？

A: 检查以下几点：

  • 确保 DirectML.dll 在程序目录中
  • 确认 GPU 驱动程序已更新到最新版本
  • 在偏好设置中确认"渲染优先级"设为 GPU 优先
  • 查看日志文件确认 DML provider 是否成功加载

Q3: 内存占用很高？

A: v1.5 已优化内存管理——F0 模型用完自动释放。如仍偏高，可尝试：关闭 GPU 加速（GPU session 会占用额外显存）、减少同时打开的轨道数量。

Q4: 修正后的音质有异常？

A: 可能原因：声码器模型文件损坏（重新下载）、Retune Speed 设置过高（降低到 30% 以下试试）、原始录音质量较差。

Q5: 如何实现自然的修音效果？

  • 使用较低的 Retune Speed（10–30%）
  • 保留原始颤音，不添加人工 Vibrato
  • 使用 LineAnchor 工具精细调整关键位置
  • 选择"曲线优先编辑"方案获得更精细的控制

Q6: VST3 插件在 DAW 中无法加载？

A: 确保 OpenTune.vst3 文件夹完整（包含 Contents/x86_64-win/ 子目录），且 ONNX 运行时 DLL 和模型文件在正确位置。在 DAW 中重新扫描 VST3 目录。

Q7: 如何切换界面语言？

A: 偏好设置 → 语言（Language），选择目标语言后界面立即切换。支持中文、英文、日文、俄文、西班牙文。
